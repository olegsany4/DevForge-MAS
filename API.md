# API / Контракты

Если используется HTTP API, минимальные контракты специфицируются здесь или генерируются из `CONTRACTS.json`.

## Пример (псевдо-описание)

- `GET /healthz` → `{ "status": "ok" }`
- `POST /run` (запуск пайплайна) → `{ "job_id": "..." }`
- `GET /jobs/{id}` → статус задачи и ссылки на артефакты
